# 🏆 MQC 模型质量生产力认证 (Model Quality Certification)

MQC 是一套系统化的模型质量评估框架，旨在模拟真实生产环境下的高负载、长上下文和复杂对话场景，验证模型的稳定性和可靠性。

当前版本：**v1.2**

---

## 📋 认证标准概览

认证由三个核心阶段组成，涵盖了从极端边界到持续压力的全方位测试。

### Stage 1: The "Qwen Standard" (上下文极端测试)
借鉴业内领先的长上下文处理标准，重点测试模型在内存边界的表现。
- **1.1 零样本/短文本**: 验证基础连接和 10-token 极短请求的响应。
- **1.2 边界阶梯测试**: 覆盖 50%、80%、95%、100% 的上下文填充率。
- **1.3 溢出测试 (105%)**: 测试模型对超出窗口大小（105%）内容的优雅处理能力（截断或处理）。
- **1.4 NIAH 大海捞针矩阵**: 分布在 2k, 8k, 16k 及模型上限等多个上下文长度，并在 10%, 50%, 90% 等多个深度插入“针”。

### Stage 2: The "1000 Requests Standard" (压力与稳定性)
模拟真实业务中的高并发和混合负载情况。
- **2.1 短文本并发爆发 (Burst)**: 连续发送 1000 次短文本请求，验证在高并发（默认 10）下的零失败率。
- **2.2 混合负载模拟 (Mixed Load)**: 80% 短文本 + 20% 长文本 (4k+) 混合并行，测试推理引擎的调度稳定性。
- **指标**: 强制计算系统级吞吐量 (Input/Output Throughput) 和 P99 延迟。

### Stage 3: Multi-round Dialogue (多轮对话一致性)
测试模型在长时会话中的语义记忆能力。
- **3.1 对话稳定性测试**: 并行运行 A, B, C 三组对话，每组默认 20 轮。
- **3.2 语义一致性检查**: 在对话开始时告知“秘密密钥”，在最后一轮要求模型复述，并使用模糊语义匹配算法进行验证。

## 🎨 ABC 内容多样化策略

为了确保测试结果不依赖于特定的语料类型，MQC 使用 ABC 三类完全不同的生成策略：
- **Version A (Code/Technical)**: 包含函数定义、JSON 结构和系统日志。
- **Version B (Narrative/Prose)**: 包含文学叙述、新闻报道和散文。
- **Version C (Data-dense)**: 包含技术规格说明书、数据表格和密集型指标。

## 🛠 核心技术特性

### 1. 精确 Token 校准 (Token Calibration)
不使用粗略的字符数或词数估算。通过 `DiverseContentGenerator.generate_calibrated` 方法，配合底层 `TokenizerService`，使用二分搜索与字符级调整，确保生成的 Prompt 长度与目标 Token 数偏差为 0。

### 2. CSV 持久化
所有 MQC 测试记录都会实时持久化到 CSV 文件中，包括专有的字段：
- `niah_correct`: 大海捞针是否命中。
- `semantic_correct`: 语义一致性是否通过。
- `is_long`: 是否为混合负载中的长文本任务。
- `system_total_throughput`: 系统整体吞入/吐出速率。

### 3. 可配置阈值
报告生成逻辑基于 `core/certification_report.py` 中的 `MQC_THRESHOLDS` 配置：
- 爆发测试成功率 P1 门槛：99.9%
- NIAH 通过率：>= 80%
- 边界测试错误数：0 容忍

## 🚀 如何运行

### UI 方式
1. 启动应用：`streamlit run app.py`
2. 在左侧侧边栏选择 **🏆 MQC 质量认证** 面板。
3. 配置最大上下文窗口和稳定性请求数。
4. (高级) 展开高级配置以调整对话轮数、并发数及 NIAH 深度。
5. 点击 **开始认证测试**。

### 产出物
- **实时日志**: Streamlit 界面显示详细阶段进度。
- **认证报告**: 自动生成 Markdown 格式的正式报告，保存在 `quality_results/certification/` 目录下。
- **原始数据**: 保存至主 CSV 日志中，支持二次深度分析。
