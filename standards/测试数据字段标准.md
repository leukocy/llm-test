# 测试数据字段标准(统一注册表)

> 目的:把 **Excel 手工模板**(模型性能测试/)与 **仓库自动采集**(hmTest/hwInventory/model_spec + per-cell 监控)并成**一份权威字段注册表**。原则:**补全不替代**——两边各有对方没有的东西,统一成一套,每个字段标清来源/Excel列/仓库列/缺口。

## 来源标签
- `自动` = benchmark/指纹/docker inspect 自动采集(不该手填)
- `手工` = 人填写(机房/拓扑/登记负责人等,机器采不到)
- `专项` = 需专门测试产(Prefill传输分解、Decode通信/计算拆分、nsys剖析)
- `推导` = 由其它字段算(等效带宽、带宽利用率、瓶颈判断)

## 缺口汇总与补全优先级

**P0(自动→手工,立刻能补)**
- 仓库的 per-cell **温度/功耗/利用率/显存**自动采集 → 导一行喂进 Excel 基础性能表(替手抄)
- Excel **加 GPU温度列**(散热问题靠它预警)+ **归因列**(status/bottleneck)
- Excel 加 **p50/p95/p99** 百分位列

**P1(专项→仓库,需扩 benchmark)**
- E 维 Prefill/Decode 瓶颈专项:做成 benchmark 可产的模板(`prefill_transport_test` / `decode_breakdown_test`),数据入仓。**这是"通信vs计算"瓶颈的归宿**。
- 仓库加 冷/热启动 TTFT 区分(跑两次:首次冷、缓存热)

**P2(登记/场景,手工为主)**
- 仓库补 model_spec 从 config.json 自动推(参数/精度/架构)——已部分做(Kimi-K2.7)
- F 维场景表:理论/估算列保持手工(评估性质),实测列接 D 维

## 治理
- 本文件 = 唯一权威字段注册表。Excel 模板与仓库字段都引用它,新增字段先入此表。
- 来源标签是硬约束:`自动`字段禁止手填,`专项`字段禁止假装自动采。
- 后续可机器化:本注册表转 YAML → 驱动 Excel 校验 + 仓库导出列顺序。
